# Jay-Board

Spring Boot 기반 게시판 프로젝트입니다.
기본 게시판 CRUD 기능에서 시작하여 회원 인증, 댓글, 좋아요, 페이징, Spring Security 기반 권한 처리, 관리자 페이지 구조까지 확장하는 것을 목표로 개발 중입니다.

---

## 1. 프로젝트 개요

Jay-Board는 게시글 작성, 조회, 수정, 삭제 기능을 중심으로 한 게시판 서비스입니다.

단순 CRUD 구현에 그치지 않고, 실제 서비스 운영에 필요한 로그인 인증, 권한 처리, 댓글, 좋아요, 관리자 페이지, 통계, 카테고리 권한 구조까지 확장하는 방향으로 개발하고 있습니다.

기존 세션 기반 로그인 구조에서 Spring Security 기반 인증 구조로 전환했습니다.

* Spring Security Form Login 적용
* 사용자 로그인 URL 분리

  * 로그인 페이지: `/member/login`
  * 로그인 처리: `/member/loginPost`
* 비로그인 사용자 접근 제어
* Ajax 요청과 일반 페이지 요청의 인증 실패 응답 분리

  * 일반 페이지 요청: 로그인 페이지로 이동
  * Ajax 요청: 401 JSON 응답
* 사용자 인증 객체를 `CustomUserDetails`로 관리
* Controller에서 기존 세션 로그인 객체 의존 제거
* URL 단위 접근 권한 관리

## 4. 관리자 페이지 설계

관리자 페이지는 일반 사용자 계정과 분리된 관리자 계정 기반으로 설계합니다.

관리자는 게시판 운영 현황을 확인하고, 회원/게시글/댓글/카테고리/통계를 관리할 수 있습니다.

---

### 4.1 관리자 계정 구조

일반 회원 테이블과 관리자 계정 테이블을 분리합니다.

* 일반 회원: `members`
* 관리자 계정: `tb_admin_member`

관리자 등급은 다음과 같이 구분합니다.

| 등급      | 설명                        |
| ------- | ------------------------- |
| MASTER  | 전체 기능 사용 가능, 관리자 계정 관리 가능 |
| MANAGER | 회원/게시글/댓글/카테고리 관리 가능      |
| VIEWER  | 조회만 가능                    |

관리자 상태는 다음과 같이 관리합니다.

| 상태      | 설명    |
| ------- | ----- |
| ACTIVE  | 사용 가능 |
| BLOCKED | 차단    |

#### 7. 관리자 관리

관리자 관리는 MASTER 등급 전용 메뉴입니다.

MANAGER, VIEWER 등급 관리자에게는 해당 메뉴를 노출하지 않으며, 직접 URL 접근도 서버에서 차단합니다.

##### 관리자 권한

| 등급      | 가능 기능                |
| ------- | -------------------- |
| MASTER  | 전체 가능, 관리자 계정 관리 가능  |
| MANAGER | 회원/게시글/댓글/카테고리 관리 가능 |
| VIEWER  | 조회만 가능               |

##### 관리자 생성

* MASTER만 관리자 계정 조회 및 생성 페이지 접근 가능
* 관리자 생성/수정/상태 변경은 운영 로그에 기록

## 5. 카테고리 기반 권한 확장 계획

추후 게시판은 카테고리 기반 구조로 확장합니다.

회원은 자신의 회원 유형에 따라 접근 가능한 카테고리만 조회하거나 작성할 수 있습니다.

### 회원 유형 예시

| 유형      | 설명    |
| ------- | ----- |
| NORMAL  | 일반 회원 |
| VIP     | 우수 회원 |
| COMPANY | 기업 회원 |

### 카테고리 접근 예시

| 카테고리   | NORMAL   | VIP      | COMPANY  |
| ------ | -------- | -------- | -------- |
| 자유게시판  | 읽기/쓰기 가능 | 읽기/쓰기 가능 | 읽기/쓰기 가능 |
| VIP게시판 | 접근 불가    | 읽기/쓰기 가능 | 접근 불가    |
| 기업게시판  | 접근 불가    | 접근 불가    | 읽기/쓰기 가능 |
| 공지사항   | 읽기 가능    | 읽기 가능    | 읽기 가능    |

서버에서는 화면 노출 여부와 별개로, 게시글 조회/작성 시 카테고리 접근 권한을 반드시 검증합니다.

## 6. 배포 방법

민감한 서버 정보는 공개 저장소에 노출하지 않기 위해 예시 값으로 표기합니다.

### 로컬 빌드

```bash
./mvnw clean package
```

Windows PowerShell 기준:

```powershell
.\mvnw.cmd clean package
```

### 서버 업로드

```powershell
scp .\target\board-0.0.1-SNAPSHOT.war <DEPLOY_USER>@<SERVER_HOST>:/path/to/deploy/ROOT.war
```

### 서버 접속

```powershell
ssh <DEPLOY_USER>@<SERVER_HOST>
```

### 배포 실행

```bash
/path/to/deploy.sh
```

### 로그 확인

```bash
tail -f /path/to/app.log
```

### 서비스 상태 확인

```bash
systemctl status <SERVICE_NAME>
```
